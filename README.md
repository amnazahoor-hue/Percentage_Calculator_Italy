# Percentuale — Online Percentage Calculator

Next.js (App Router) site in English for calculating percentages, markups, discounts, and percent change.

## Requirements

- Node.js 20+
- npm

## Local development

```bash
cd percentuale
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Environment variables

Create `.env.local` in the project folder:

```env
NEXT_PUBLIC_SITE_URL=https://yourdomain.com
NEXT_PUBLIC_GA_ID=G-XXXXXXXXXX
NEXT_PUBLIC_CLARITY_ID=xxxxxxxxxx
NEXT_PUBLIC_GSC_VERIFICATION=google-site-verification-code
```

| Variable | Description |
|----------|-------------|
| `NEXT_PUBLIC_SITE_URL` | Canonical site URL (sitemap, Open Graph, JSON-LD) |
| `NEXT_PUBLIC_GA_ID` | Google Analytics 4 — leave empty to disable |
| `NEXT_PUBLIC_CLARITY_ID` | Microsoft Clarity — leave empty to disable |
| `NEXT_PUBLIC_GSC_VERIFICATION` | Google Search Console verification code |

## Deploy (Vercel recommended)

1. Import the repository on [Vercel](https://vercel.com).
2. Set the root directory to `percentuale` if the repo includes a parent folder.
3. Add the environment variables above.
4. Build command: `npm run build` — Output: default Next.js.

HTTPS is handled automatically on Vercel.

## Main structure

- `src/app/` — pages and routes (home, legal, contact, sitemap, robots)
- `src/components/calc/` — 3D neumorphic calculator (hero)
- `src/components/layout/ThemeToggle.tsx` — light/dark toggle
- `src/lib/calc.ts` — calculator logic
- `src/lib/theme.ts` — theme provider + `localStorage`
- `src/styles/globals.css` — light/dark design tokens (CSS variables)

## Scripts

- `npm run dev` — development
- `npm run build` — production build
- `npm run start` — production server
- `npm run lint` — ESLint

## Notes

- The contact form uses a honeypot anti-spam field; connect a backend or email service in production.
- The Open Graph image is generated by `src/app/opengraph-image.tsx`.
- Respect `prefers-reduced-motion` for Framer Motion animations.
